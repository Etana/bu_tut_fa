Nouvelle fonctionnalité : Choix entre mails et/ou notifications

 par Vivi le 2013-08-19, 15:45

Chers utilisateurs,


Nous avons le plaisir de vous présenter une nouvelle fonctionnalité basée sur vos demandes . Celle-ci va vous permettre de choisir le type de notification que vous désirez (notifications par mails vs notifications via votre Toolbar).


Remarque importante:

Les administrateurs ayant des templates personnalisés, doivent obligatoirement apporter les modifications ci-dessous en Punbb ou en PhpBB2 a leurs templates afin de bénéficier de cet ajout
Où se trouve cette nouvelle fonctionnalité ?


Sur votre profil dans l'onglet "Notifications" :



Notifications dans la Toolbar :


Dans ce cas, le forum notifiera lutilisateur uniquement via le système de notification de la ToolBar et rien dautre. Aucun email de notification pour les 8 éléments concernés par ces alertes ne sera envoyé au concerné.


Notifications par email : 

Dans ce cas, le forum notifiera lutilisateur uniquement via un envoi demail de notification et rien dautre. Aucune notification pour les 7 éléments concernés par ces alertes ne sera affichée via le système de notification de la ToolBar.
 La possibilité dêtre averti par mail lors de la connexion d'un ami ne sera pas possible pour éviter d'être spammer par vos forums


Les deux :

Dans ce cas, le forum notifiera lutilisateur via le système de notification de la ToolBar ainsi que via un envoi demail de notification. Ce fonctionnement est déjà opérationnel en ligne car cest celui adopté à la création du système de notifications.

Par défaut, cette fonctionnalité sera sur "Les deux". Cela, afin de ne pas perturber le fonctionnement actuel en ligne et les habitudes de nos utilisateurs.

Cependant pour les options suivantes les notifications par email seront décochées par défaut :

-Les demandes d'amitiés
-La réception d'un message sur le profil
-Un abus signalé


Si ladmin désactive la ToolBar du forum, le choix des notifications revient à celui par défaut. 

Bouton "Tout supprimer" :

Nous avons également ajouté un nouveau bouton de suppression de masse pour les notifications enregistrées dans le centre de notifications.



N.B :
Pour éviter les doublons, nous avons supprimé deux options dans les préférences de profil utilisateur comme ils sont actuellement gérés directement à partir de l'onglet Notifications.
Ce sont:
- M'avertir des nouveaux Messages Privés
- Être averti par mail si un nouveau rapport est posté


En éspérant que cette fonctionnalité sera à la hauteur de vos attentes

Nos membres ayant des Templates personnalisés avec la versions Phpbb2 vont devoir effectuer quelques modifications

Modifications à annoncer sur les templates persos :

Template profile_add_body dans Profil

Il faut supprimer :

Code:
<tr>
  <td class="row1"><span class="gen">{L_NOTIFY_ON_PRIVMSG} :</span></td>
  <td class="row2">
    <label class="gen"><input type="radio" name="notifypm" value="1" {NOTIFY_PM_YES} />{L_YES}</label>  
    <label class="gen"><input type="radio" name="notifypm" value="0" {NOTIFY_PM_NO} />{L_NO}</label>
  </td>
</tr>
Situé après :

Code:
<tr>
  <td class="row1"><span class="gen">{L_NOTIFY_ON_REPLY} :</span><br /><span class="gensmall">{L_NOTIFY_ON_REPLY_EXPLAIN}</span></td>
  <td class="row2">
    <label class="gen"><input type="radio" name="notifyreply" value="1" {NOTIFY_REPLY_YES} />{L_YES}</label>  
    <label class="gen"><input type="radio" name="notifyreply" value="0" {NOTIFY_REPLY_NO} />{L_NO}</label>
  </td>
</tr>
Il faut supprimer :
Code:
<tr>
  <td class="row1"><span class="gen">{switch_preferences_menu.switch_report.L_NO_REPORT_MAIL} :</span></td>
  <td class="row2">
    <label class="gen"><input type="radio" name="no_report_mail" value="0" {switch_preferences_menu.switch_report.NO_REPORT_MAIL_YES} />{L_YES}</label>  
    <label class="gen"><input type="radio" name="no_report_mail" value="1" {switch_preferences_menu.switch_report.NO_REPORT_MAIL_NO} />{L_NO}</label>
  </td>
</tr>
Situé après :

Code:
<tr>
  <td class="row1"><span class="gen">{switch_preferences_menu.switch_report.L_NO_REPORT_POPUP} :</span></td>
  <td class="row2">
    <label class="gen"><input type="radio" name="no_report_popup" value="0" {switch_preferences_menu.switch_report.NO_REPORT_POPUP_YES} />{L_YES}</label>  
    <label class="gen"><input type="radio" name="no_report_popup" value="1" {switch_preferences_menu.switch_report.NO_REPORT_POPUP_NO} />{L_NO}</label>
  </td>
