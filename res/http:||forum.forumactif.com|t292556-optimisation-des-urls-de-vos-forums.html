Optimisation des URLs de vos forums

 par The Godfather le Mer 15 Déc 2010 - 11:44

Chers utilisateurs et membres,

Afin d’améliorer le référencement de vos forums et de simplifier leur utilisation, nous vous informons que des changements vont avoir lieu au niveau des URLs des pages internes de vos forums.

Bien que cette transition soit transparente pour vos forums, elle apportera un coup de pouce au référencement de vos pages dont l'architecture a été revue et améliorée. Il en découle des URLs plus courtes et très facilement indéxable. Bien évidemment, les anciennes URLs seront automatiquement redirigées vers les nouvelles.

Exemples de modification d'URL:

  • index du forum:

    http://forum.forumactif.com/forum.htm >> http://forum.forumactif.com/forum

  • portail du forum:

    http://forum.forumactif.com/portal.htm >> http://forum.forumactif.com/portal

  • un sujet:

    http://forum.forumactif.com/forum-f71/titre-t15.htm >> http://forum.forumactif.com/t15-titre


...etc.

Néanmoins, et dans le cas improbable de l’apparition de problèmes, notre équipe est à votre écoute sur ce forum de support.

L’équipe Forumactif



Remarque importante: Afin que ces modifications soient prises en compte, les membres ayant des forums sous les versions PhpBB2 & PunBB avec des templates personnalisées doivent absolument prendre en considération les modifications ci-dessous au cas ou ils auraient modifié les templates citées.



phpbb2 & punbb

¤ Description : Suppression du "/" écrit en dur avant les liens vers /statistics, /donate et /contact


  • Dans overall_footer,

    remplacer :

    Code:

    href="/{switch_view_stats.S_STATS}"

    par :

    Code:

    href="{switch_view_stats.S_STATS}"


    remplacer :

    Code:

    href="/{switch_view_donation.U_DONATION}"

    par :

    Code:

    href="{switch_view_donation.U_DONATION}"


    remplacer :

    Code:

    href="/{switch_view_contact.U_CONTACT}"

    par :

    Code:

    href="{switch_view_contact.U_CONTACT}"



¤ Description : Ajout des URL canoniques sur les pages /login et les pages de sujets


  • Dans overall_header,

    avant :

    Code:

       {META_FAVICO}


    ajouter :

    Code:

       <!-- BEGIN switch_canonical_url -->
       <link rel="canonical" href="{switch_canonical_url.CANONICAL_URL}" />
       <!-- END switch_canonical_url -->





¤ Description : remplacement des url écrites en dur dans les templates


  • Dans album_showpage_body

    remplacer :

    Code:

    album_showpage.forum

    par :

    Code:

    {U_ALBUM_SHOWPAGE}


  • Dans profile_view_body

    remplacer :

    Code:

    ajax_profile.forum

    par :

    Code:

    {U_AJAX_PROFILE}


  • Dans search_body

    remplacer :

    Code:

    googleresults.forum

    par :

    Code:

    {U_GOOGLERESULTS}


  • Dans posting_body

    remplacer :

    Code:

    faq.htm

    par :

    Code:

    {U_FAQ}


  • Dans album_upload_body

    remplacer :

    Code:

    album_nuffload_pbar.php

    par :

    Code:

    {U_ALBUM_NUFFLOAD}





punbb

¤ Description : remplacement du lien vers les FAQ écrit en dur dans les templates


  • Dans posting_body_wysiwyg

    remplacer :

    Code:

    faq.htm

    par :

    Code:

    {U_FAQ}






Edit le 22/12/2010:

chose promise chose due

nous venons de publier sur vos forum une mise à jour vous permettant de controler la cible du lien dans la colonne « derniers messages » sur l'index

Cette modification concerne la colonne « derniers messages » sur l’index. Actuellement, le lien du sujet dirige vers le 1er post du sujet et l’icône « voir le dernier message » vers le dernier message du sujet.

Afin de satisfaire le plus grand nombre d'entre vous, nous avons fait en sorte que la modification du fonctionnement de ces liens soit accessible aux admins via les templates afin qu'ils choisissent eux même le fonctionnement voulu.

voici les modifications a faire:

Version du forum: PhpBB2 & PunBB

¤ Description : Permet de personnaliser les liens des derniers sujets de la page d'accueil.

Pour les admins ayant des templates personnalisé:


  • Dans le templateindex_box,

    remplacer :

    Code:

    {catrow.forumrow.LAST_POST}

    par (identique à la nouvelle version de l'index) :

    Code:


                <span>
                <!-- BEGIN switch_topic_title -->
                <a href="{catrow.forumrow.U_LATEST_TOPIC}" title="{catrow.forumrow.LATEST_TOPIC_TITLE}">{catrow.forumrow.LATEST_TOPIC_NAME}</a>

                <!-- END switch_topic_title -->
                {catrow.forumrow.USER_LAST_POST}
                </span>
             

    ou par (équivalent de l'ancienne version de l'index) :

    Code:


                <span>
                <!-- BEGIN switch_topic_title -->
                <a href="{catrow.forumrow.U_LATEST_TOPIC_POST}" title="{catrow.forumrow.LATEST_TOPIC_TITLE}">{catrow.forumrow.LATEST_TOPIC_NAME}</a>

                <!-- END switch_topic_title -->
                {catrow.forumrow.USER_LAST_POST}
                </span>
             



Pour les admins n'ayant pas modifier de templates:


  • Dans le template index_box,

    remplacer :


    Code:

    {catrow.forumrow.U_LATEST_TOPIC}


    par (équivalent de l'ancienne version de l'index) :


    Code:

    {catrow.forumrow.U_LATEST_TOPIC_POST}





Spharhatdhrat a écrit:Bonsoir,

Après observation, je constate que vous avez une ancienne version du template index_box qui contient des lignes obsolètes. Afin d'appliquer la modification vous pouvez :

1. si vous n'avez pas d'autres modifications dans ce template, supprimer et rééditer votre template en faisant uniquement la modification suivante :

remplacer :

Code:

{catrow.forumrow.U_LATEST_TOPIC}

par :

Code:

{catrow.forumrow.U_LATEST_TOPIC_POST}


2. si vous avez des modifications que vous souhaitez conserver :

supprimer la ligne :

Code:


      <!-- BEGIN forum_link_no -->

supprimer le bloc suivant :

Code:


      <!-- END forum_link_no -->
      <!-- BEGIN forum_link -->
      <td class="row3" colspan="3" align="center" valign="middle" height="50"><span class="gensmall">{catrow.forumrow.forum_link.HIT_COUNT}</span></td>
      <!-- END forum_link -->
